# Задание 3 — Архитектура PUSH уведомлений

**Описание:**  
Мобильное приложение "Петрушка Зеленая" должно получать различные push-уведомления:  
- уведомления о забытой корзине,  
- уведомления об отмене заказа,  
- рекламные пуши и др.

**Задача:**  
Построить верхнеуровневую архитектурную схему отправки PUSH уведомлений с учётом микросервисной архитектуры на бэкенде.

## Схема

Схема находится в файле [`push_architecture.png`](push_architecture.png)

### Основные компоненты:

- **Мобильное приложение** — принимает пуши через FCM / APNs  
- **API Gateway / Backend** — принимает события от микросервисов  
- **Микросервисы** — Cart Service, Order Service, Marketing Service  
- **Event Broker / Message Queue** — Kafka / RabbitMQ / SQS  
- **Push Notification Service** — отправляет уведомления на устройства пользователей
